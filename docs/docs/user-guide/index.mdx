# User Guide Overview

`tilearn` currently focuses on practical **single-machine scheduling** workflows.
This section explains which algorithm to choose, how to prepare data, and how
to map theory to executable Python code.

## Core objective families

### 1) Minimize maximum lateness: $L_{\max}$

Use **EDD (Earliest Due Date)** ordering when due-date compliance is your
primary concern.

- Primary guide: [EDD](/user-guide/single-machine/lateness/edd)
- API entry point: [`tilearn.edd`](/api-reference/modules/sorting#tilearnedd)

### 2) Minimize weighted completion behavior: $\sum w_j C_j$

Use **WSPT** ordering to prioritize higher-weight, shorter-processing jobs.

- Primary guide: [WSPT](/user-guide/single-machine/completion/wspt)
- API entry point: [`tilearn.wspt`](/api-reference/modules/sorting#tilearnwspt)

### 3) Mixed-list orchestration with precedence flags

Use the multi-list pipeline when your workload combines:

- independent lists (`prec=0`), and
- precedence-style lists (`prec=1`).

- End-to-end guide: [Illustrative workflow](/user-guide/illustrate)
- API entry point: [`tilearn.optimal_list`](/api-reference/modules/joblist)

## Recommended reading order

1. Start with [Getting started](/getting-started).
2. Read either [EDD](/user-guide/single-machine/lateness/edd) or
   [WSPT](/user-guide/single-machine/completion/wspt) based on your objective.
3. Move to [Illustrative workflow](/user-guide/illustrate) for multi-file
   production-like pipelines.
4. Use [API reference](/api-reference) for parameter-level details.
