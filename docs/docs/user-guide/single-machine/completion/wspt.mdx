# Weighted Completion Time with WSPT

## WSPT algorithm

For single-machine jobs with objective $\sum w_j C_j$, the classic dispatching
rule is to process jobs in descending order of:

$$
\frac{w_j}{p_j}
$$

where:

- $w_j$ is job weight (priority),
- $p_j$ is processing time.

When all weights are equal, WSPT reduces to SPT.

## API used in `tilearn`

- Main function: [`tilearn.wspt`](/api-reference/modules/sorting)
- Supporting helper: [`tilearn.factor_data`](/api-reference/modules/basis)

## Input format

Each row follows:

```python
[name, p, r, d, w]
```

Example:

```python
jobs = [
    ["J1", 2.0, 0, 8, 4.0],
    ["J2", 1.0, 0, 5, 1.0],
    ["J3", 3.0, 0, 6, 6.0],
]
```

## Minimal runnable example

```python
import tilearn as tl

jobs = [
    ["J1", 2.0, 0, 8, 4.0],
    ["J2", 1.0, 0, 5, 1.0],
    ["J3", 3.0, 0, 6, 6.0],
]

ranked = tl.wspt(jobs)
for row in ranked:
    print(row)
```

Output shape:

```text
[name, p, r, d, w, factor]
```

The appended `factor` column is `w/p`, and rows are sorted in descending
factor order.

## Practical notes

- `wspt` mutates the input table in-place.
- Calling it repeatedly on the same data may append additional factor columns.
- Use a copy when you need to preserve original rows:

```python
safe_jobs = [row[:] for row in jobs]
ranked = tl.wspt(safe_jobs)
```
